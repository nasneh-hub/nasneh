## ğŸ” Component-Specific Notes

### 1. Button
- Variants: default (black), secondary (gray), ghost, destructive
- Icon button uses `rounded-full` (exception to rounded-xl rule)
- Loading state with spinner

### 2. Input
- States: default, focus (ring), error (red bg), success (green bg)
- NO borders â€” use background colors
- Support for prefix/suffix icons

### 3. Card
- NO borders â€” use shadow-sm
- Consistent padding (var(--space-4))
- Optional header/footer sections

### 4. Badge
- Semantic variants: success, warning, danger, info, default
- Small size only
- NO borders

### 5. Dialog
- Overlay with backdrop
- Close button (X)
- Header, content, footer sections
- Trap focus for accessibility

### 6. SegmentedControl
- Use for 2-6 options ONLY
- 7+ options = use Select instead
- Active segment has background change

### 7. Table
- NO row borders â€” use hover:bg or alternating backgrounds
- Sortable headers (optional)
- Responsive behavior (horizontal scroll on mobile)
- Consistent cell padding

### 8. Select
- For 7+ options
- Search/filter support for long lists
- NO borders â€” use background
- Dropdown uses shadow-md

### 9. Tabs
- Horizontal layout
- Active tab indicator (bottom line or background)
- NO borders on tabs

### 10. Avatar
- Sizes: sm (32px), md (40px), lg (56px)
- Fallback to initials
- rounded-full always
- Optional status indicator

### 11. Toast
- Position: top-right
- Auto-dismiss (5 seconds default)
- Variants: success, error, warning, info
- Close button optional

### 12. Skeleton
- Animate with pulse
- Match component shapes (rectangle, circle, text lines)
- Use for loading states
